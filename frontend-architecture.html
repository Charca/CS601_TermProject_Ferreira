Building Resilient Frontend Architecture

These are my notes on Monica Lent's talk Building Resilient Frontend Architecture.

Think of architecture as enabling constraints.

3 constraints for a more resilient front-end architecture:

  1. Source code dependencies must point inwards. Use levels of dependencies (layered) instead of a big ball of mud with circular dependencies.
  2. Be conservative about code reuse. For shared components, stick them into the design system, or copy-paste them. Decoupled code is better than DRY code.
  3. Enforce your boundaries. Use automated tools. For example, forbidden dependency test in CI to prevent importing modules from other apps (`dependency-cruiser` in npm.)

Final takeaway:

You don't have to derive architecture from first principles. Be open-minded and open to learning from other paradigms, languages and communities.
