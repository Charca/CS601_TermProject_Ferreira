React 18 useTransition Hook

The startTransition function is one of the new APIs in React 18 that is also one of the new concurrent features (remember, there's no concurrent mode anymore, only concurrent features).

startTransition lets us define a state update (useState, useReducer, this.setState) as a "transition", which means a non-urgent update.

In React 18, there are two types of state updates:

  • Urgent updates: the default type, reflects a direct user interaction like typing or clicking.
  • Transition updates: a non-urgent update which transitions the UI from one state to the next like updating a list of results based on a filter value. These have lower priority and so their rendering can be interrupted.

startTransition vs useTransition

There is also a useTransition hook which returns a startTransition instance. They work very similarly but useTransition returns a flag that reflects if the transition is pending. This can be used as a "loading flag".

As a rule of thumb, if you care about the value of `isPending` and you're using it in a React component, use the `useTransition` hook, otherwise, stick to `startTransition`.
